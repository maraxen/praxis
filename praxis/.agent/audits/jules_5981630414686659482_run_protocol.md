# Jules Audit: Run Protocol Feature
**Session ID**: 5981630414686659482
**Date**: Feb 3, 2026
**Quality Score**: 4/10

## Key Findings

### Simulation vs Physical Execution Paths
The application maintains strict separation between simulation and physical execution:
- **UI Guard**: `showMachineError` prevents advancing in Physical Mode with simulated hardware
- **Service Guard**: `ExecutionService.startRun` has defense-in-depth check

### Critical Issues
1. **Browser Mode Machine Config**: `resolved_assets_json` is null and never updated
2. **Ephemeral Resource Cleanup**: No logic to purge `is_ephemeral` resources
3. **Linear Stepper Brittleness**: Hardcoded `selectedIndex = 2` for snap-back
4. **Simulator Template Discovery**: Fallbacks and console.warn in `configureSimulationTemplate`

## Recommendations
1. Populate `resolved_assets_json` in `startBrowserRun`
2. Implement cleanup job or "Purge Ephemeral" action
3. Use named step mapping for stepper references
4. Formalize simulator backend mapping in schema
